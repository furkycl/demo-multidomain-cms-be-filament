# i18n — Multi-locale architecture

## Desteklenen 10 dil

| Code | Native | CRM target |
|------|--------|-----------|
| tr | Türkçe | Omnigos |
| en | English | Omnigos |
| ar | العربية (RTL) | Omnigos |
| fr | Français | Linguland |
| es | Español | Linguland |
| pt | Português | Linguland |
| ko | 한국어 | Linguland |
| ja | 日本語 | Linguland |
| it | Italiano | Linguland |
| de | Deutsch | Linguland |

Tek kaynak: `config/locales.php`. Frontend bu listeyi `GET /api/sites/{domain}` endpoint'inden çeker.

## Veri modeli

`pages` tablosunda `locale` kolonu. Aynı `(site_id, slug)` çiftine **farklı locale ile birden fazla** Page kaydı düşebilir — her biri o dilin çevirisi.

Unique constraint: `(site_id, locale, slug)`.

## URL yapısı

`https://kaplan-london.com/[locale]/[slug]`

- `/` → `/tr` (default locale, 308 redirect)
- `/tr/` → home
- `/tr/courses` → courses page
- `/en/courses` → courses page in English
- `/ar/courses` → RTL Arabic version

## hreflang

`SiteController::showPage` yanıtında `alternates` map'i döner — frontend bunu `<link rel="alternate" hreflang="..." href="..." />` olarak head'e basar.

## Yeni page oluşturma akışı

1. Filament → Pages → Create
2. Site seç, **locale seç (10 dilden biri)**, slug yaz
3. Bloklar sekmesinden bloklar ekle
4. Diğer diller için aynı slug ile yeni Page kaydı + içerikler

İleride: "Page'i bütün dillere kopyala" Filament Action'ı eklenebilir (boş çeviri kayıtları üretir, admin sonradan doldurur).
